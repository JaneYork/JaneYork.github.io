001 - 安装与使用

发表于 2020-04-01 更新于 2020-04-03 分类于 建站 阅读次数：
本文字数： 780 阅读时长 ≈ 1 分钟

前情提示

服务器：西数云服务器；

系统：CentOS Linux release 7.4.1708 (Core) ；

Git：1.8.3.1；

nodejs：v13.12.0;

前置条件

  1. Git安装完成，详细步骤请移步：https://janeyork.blog.csdn.net/article/details/103007119
  2. nodejs安装完成（版本不低于8.10），详细步骤请移步：https://mp.csdn.net/console/editor/html/82690129

安装Hexo

  1. 官网地址悉知：https://hexo.io/zh-cn/

  2. 输入npm install -g hexo-cli来进行安装。此步骤，请耐心等待。

  3. 添加环境变量

4.安装成功后输入hexo -v进行测试。

  1. 进入cd /home，然后创建你的第一个站点，比如叫做myblog输入命令hexo init myblog（耐心等待2分钟）

  2. 进入cd myblog，安装依赖npm install。

  1. 生成静态文件hexo g。
  2. 启动服务hexo g默认是监听4000端口，直接访问http://yourip:4000即可。

注意：如果不能访问请排查防火墙或者云服务器安全组问题。

例如：http://west.nogit.top:4000/

西数服务器个人感觉真难用，无法直接使用IP访问，其他平台域名解析IP后，添加白名单，开放安全组，CNAME解析，还是无法访问，还需要漫长的工单等待。而且工单只能提交一个。而且我就呵呵笑了，提工单还需要money，当然了，我个人不反对！

解决：

西数服务器centOS7，其实有两个防火墙，另一个iptables也需要关闭下。

  1. 成功后浏览器访问，Hexo默认主题。

总结

  1. 托管GitHub,使用子域名githu.io逼格较高，较为流行，支持自动推送部署。
  2. 访问速度相对较慢，相对于coding、码云，毕竟国内。
  3. 如果想要绑定自己自定义域名，自定义域名无法SSL，即便采用第三方DNS方代理，貌似也是牺牲@主域名。
  4. 理论上每个用户只能有一个类似username.github.io域名，如果想要多站点，需要gh-pages分支，访问d以username.github.io/repo1。

代替方案

  1. coding：支持多域名、多站点、速度较快、自定义域名可以强制HTTPS。
  2. 码云：速度较快、如果想要自定义域名、推送自动部署、强制HTTPS，99￥/年，谢谢！域名访问暂不清楚是否支持自定义域名，如果是码云默认格式：http://janeyork.gitee.io/blog/ 说实在，我没有仔细研究，我的样式为什么没了，同样是推送到其他平台就不会有问题。
  3. Netlify。
  4. 其他平台。
